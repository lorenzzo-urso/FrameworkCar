## Mapa de valor

### Aliviadores de dores
| Dor | Como o Compadre alivia | Encaixe |
|---|---|---|
| Não entende APP/RL | Camada semântica explica e pré-preenche via SIGEF/MapBiomas | ✅ |
| Notificação incompreensível | Traduz em "o quê / por quê / como corrigir" | ✅ forte |
| Medo de declarar errado | Validação antecipada cruza declaração × dados abertos | ✅ |
| Não usa o SICAR sozinho | Traduz o que cada etapa pede em linguagem simples + abre deep link direto na etapa certa do SICAR quando o Raimundo está pronto | ✅ |
| Vai-e-vem de retificações | Usa a **mesma base de referência do SFB** que o analista do OEMA usa para validar — o CAR já chega certo na origem, sem divergência de base | ✅ forte |
| Não sabe dos benefícios | Mostra os programas que o CAR habilita | ✅ diferencial |
| Internet limitada | Canal leve (WhatsApp em 2G) | ⚠️ parcial |

### Geradores de ganho
| Ganho | Como cria | Encaixe |
|---|---|---|
| CAR sem erro sem despachante | Pré-preenchimento + guia | ✅ |
| Processo rápido pelo celular | Fluxo conversacional curto | ✅ |
| Tranquilidade contra fiscalização | Diagnóstico claro + validação | ✅ |
| Saber o que o CAR desbloqueia | Painel de programas elegíveis | ✅ âncora ⭐ |

## Top 3 encaixes para o pitch

1. *"Raimundo recebe um aviso do governo que parece outra língua. Nós traduzimos —
   e dizemos exatamente o que fazer."* (notificação → camada semântica)
2. *"Ele não precisa saber o que é APP. Os dados abertos já sabem — ele só confirma
   no mapa."* (pré-preenchimento SIGEF/MapBiomas)
3. *"Ele entra para evitar multa. Sai sabendo que pode acessar crédito e ser pago
   por preservar."* ⭐ âncora

## Encaixes fracos (honestidade)

- **Internet inexistente:** WhatsApp ajuda, mas não resolve ausência total de
  sinal. Ponte: CATI/cooperativa presencial.
- **Substituir intermediário:** a solução **empodera** o CATI, não o elimina.
